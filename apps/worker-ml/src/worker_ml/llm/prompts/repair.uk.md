Виправ попередню відповідь моделі так, щоб вона стала валідним JSON за схемою.

Твоє завдання — лише JSON-repair. Не переосмислюй статтю, не повторюй основне завдання, не оцінюй відповідь заново і не додавай нових фактів. Збережи той самий зміст настільки точно, наскільки це можливо, але виправ формат, типи, назви полів, структуру об'єктів і відсутні обов'язкові поля.

Схема є єдиним джерелом істини. Поверни JSON, який суворо відповідає схемі.

Правила виправлення:

* Поверни лише JSON-об'єкт або JSON-масив, залежно від того, що вимагає схема.
* Не повертай markdown, пояснення, коментарі або текст поза JSON.
* Не повертай JSON як рядок.
* Виправ неправильні назви полів на назви зі схеми, якщо відповідність очевидна.
* Виправ типи значень, якщо це можна зробити без зміни змісту.
* Якщо поле є обов'язковим і його можна вивести з невалідної відповіді, додай його.
* Якщо обов'язкове поле не можна вивести без вигадування, використай найбезпечніше порожнє значення, дозволене схемою: `null`, `[]`, `{{}}` або порожній рядок — лише якщо такий тип дозволений схемою.
* Якщо неможливо зробити елемент валідним без вигадування факту або enum-значення, видали цей елемент зі списку.
* Не вигадуй значення enum. Використовуй лише значення, явно дозволені схемою.
* Якщо список перевищує максимум, залиш лише найцентральніші елементи в межах максимуму.
* Якщо об'єкт містить поля, яких немає у схемі, видали їх.
* Якщо значення суперечить схемі, але його можна безпечно нормалізувати, нормалізуй його.
* Якщо значення неможливо нормалізувати без зміни змісту, видали відповідний елемент або заміни поле на найбезпечніше дозволене порожнє значення.

Спеціальне правило для картки статті:

Якщо `is_case_candidate = false`, обов'язково поверни:

```json
{{
  "main_event_title_uk": null,
  "events": [],
  "entities": [],
  "case_signature_terms": []
}}
```

Не залишай події, сутності або case-signature terms для некандидатної статті.

Поверни лише валідний JSON за схемою.

Схема:

{schema_json}

Помилка валідації:

{validation_error}

Невалідна відповідь:

{invalid_output}
